# 项目经历（简历版）

---

## 马铃薯病害智能诊断系统

**技术栈：** Python / Flask / YOLOv8 / OpenCV / LLM (mimo-v2.5-pro) / RAG / OpenWeatherMap API / Chart.js

**项目描述：**
基于深度学习与大语言模型的农业病害检测与诊断平台。用户上传马铃薯叶片照片后，系统自动完成病害识别、环境风险评估、用药建议生成及动态防治任务编排，实现从"发现问题"到"解决问题"的全流程闭环。

---

### 核心工作与技术亮点

**1. 病害检测模块**
- 基于 YOLOv8n 训练目标检测模型，使用 Ultralytics 框架，100 epochs，图像尺寸 640×640
- 自定义可视化管线：使用 OpenCV 实现半透明填充框（15% 透明度）、类别配色、Top-K 过滤，替代 ultralytics 默认 `plot()` 方法
- 设计标签归一化层，支持中文名、拉丁学名等多别名映射，确保检测标签与知识库一致

**2. 多因子风险评估引擎**
- 设计加权评分模型（0-100 分），综合 5 个维度：病斑数量（0-40）、检测置信度（0-20）、环境湿度（0-25）、降雨状态（+20）、温度适配度（15-20）、生育期脆弱度（10-15）
- 规则引擎输出三级风险等级（低/中/高），驱动后续治疗方案差异化

**3. RAG + LLM 智能问诊**
- 构建本地农药知识库（6 种药剂，含成分、用法、用量、安全间隔期等结构化字段），通过 RAG 检索注入 LLM 上下文
- 调用 mimo-v2.5-pro 大模型，以"农业植保专家"角色生成结构化治疗建议，约束输出长度 ≤180 字，禁止编造药剂
- 实现多阶段 JSON 解析器（去 markdown 围栏 → 直接解析 → 括号深度匹配），鲁棒处理 LLM 输出噪声

**4. 动态工作流引擎**
- 基于 SOP 知识库初始化标准化防治任务序列，每步完成后农户反馈（效果明显/无改善/恶化/遇到困难）
- LLM 根据反馈历史动态生成下一步方案：正向反馈推进流程，负向触发强化措施或应急预案
- 工作流支持自适应终止（LLM 返回 `is_final: true`）

**5. 工程化能力**
- 集成 OpenWeatherMap API 获取实时气象数据（温度/湿度/降雨），覆盖 20+ 城市，含降级兜底策略
- 实现 Token 用量监控：每次 LLM 调用记录 token 消耗、延迟，Chart.js 可视化趋势仪表盘
- Flask 主流程编排完整管线：图片上传 → 检测 → 归一化 → 知识查询 → 气象获取 → 风险评分 → LLM 生成 → 页面渲染

### 可量化的成果（可自行补充）

- 训练数据集：623 张标注图像（492 训练 / 131 验证），覆盖 2 类病害 + 健康样本
- 模型体积：~6MB（YOLOv8n），适合边缘部署
- 端到端延迟：单张图片从上传到生成完整诊断报告 < 5 秒（视网络状况）
