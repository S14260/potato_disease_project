# 问题总结与解决方案

## 1. LLM 输出为空

**现象**: mimo-v2.5-pro 返回空内容

**原因**: max_tokens 设置过小（如 512），思考模型的内部推理消耗了所有 tokens，没有剩余给最终输出

**解决**: 将 max_tokens 设为 2000

```python
llm = ChatOpenAI(
    model="mimo-v2.5-pro",
    max_tokens=2000,  # 关键：思考模型需要更多 tokens
)
```

---

## 2. YOLO 重复调用

**现象**: 同一张图片被 YOLO 推理两次，浪费时间

**原因**: 原始架构中 Supervisor 的 Diagnosis Agent 会重新调用 YOLO，而 app.py 也调用了一次

**解决**: 快速模式下直接调用函数获取结构化数据，跳过 Agent 的 YOLO 调用

---

## 3. LLM 返回 JSON 格式不稳定

**现象**: 工作流场景下 LLM 返回的 JSON 可能带 markdown 代码块标记

**原因**: LLM 倾向于用 ```json 包裹 JSON 内容

**解决**: 三层解析策略
1. 直接 `json.loads` 解析
2. 去掉 ``` 标记后解析
3. 用括号匹配提取第一个完整 JSON 对象

---

## 4. ChromaDB 首次启动慢

**现象**: 第一次运行 `python rag/ingest.py` 时需要下载 Embedding 模型

**原因**: HuggingFace 模型 `BAAI/bge-small-zh-v1.5` 约 100MB，首次需要下载

**解决**: 首次运行后模型会缓存到本地，后续启动无需重复下载

---

## 5. Agent 模式响应慢

**现象**: Agent 模式下一次诊断需要 15 秒以上

**原因**: 每个 Agent 内部有 ReAct 循环（思考→调工具→再思考），加上 Supervisor 的 LLM 调用，总共 5-8 次 LLM 调用

**解决**: 默认使用快速模式（1 次 LLM 调用，3 秒内完成），Agent 模式仅用于演示

---

## 6. 前端诊断历史不更新

**现象**: Dashboard 页面数据为空

**原因**: 首次打开时没有诊断记录

**解决**: 进行几次诊断后，数据会自动填充到 `logs/diagnosis_history.json`，Dashboard 从该文件读取
