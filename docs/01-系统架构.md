# 系统架构设计

## 总体架构

马铃薯病害智能诊断系统采用 **Multi-Agent 协作架构**，基于 LangGraph 框架实现。系统将传统的单体管道拆分为四个协作的智能 Agent，通过 LLM Function Calling 实现自主任务分派和执行。

## 核心模块

### 1. Supervisor Agent（编排层）
- **文件**: `agents/supervisor.py`
- **职责**: 接收用户请求，协调三个子 Agent 按顺序执行
- **技术**: LangGraph `create_supervisor` + `create_react_agent`
- **决策方式**: LLM 自主决定调用哪个子 Agent，传入什么参数

### 2. Diagnosis Agent（诊断层）
- **文件**: `agents/diagnosis_agent.py`
- **职责**: 从叶片图像识别病害，查询病害详细信息
- **工具**: `detect_disease` (YOLO), `get_disease_info` (知识库)

### 3. Risk Agent（风险层）
- **文件**: `agents/risk_agent.py`
- **职责**: 获取气象数据，计算多因子风险评分
- **工具**: `get_weather` (OpenWeatherMap), `compute_risk_score` (规则引擎)

### 4. Treatment Agent（治疗层）
- **文件**: `agents/treatment_agent.py`
- **职责**: 检索知识库，生成结构化治疗建议
- **工具**: `search_knowledge_base` (ChromaDB RAG), `get_sop`, `generate_treatment_advice`

### 5. RAG 向量检索层
- **文件**: `rag/vectorstore.py`, `rag/retriever.py`, `rag/ingest.py`
- **职责**: 将知识库数据嵌入 ChromaDB，提供语义检索能力
- **技术**: ChromaDB + HuggingFace Embeddings (`BAAI/bge-small-zh-v1.5`)

## 数据流

```
用户上传图片 + 城市 + 生长期
        │
        ▼
Supervisor 接收请求
        │
        ├──→ Diagnosis Agent
        │    ├── detect_disease(image_path) → YOLO 检测结果
        │    └── get_disease_info(disease_id) → 病害详情
        │
        ├──→ Risk Agent
        │    ├── get_weather(city) → 温度/湿度/降雨
        │    └── compute_risk_score(detections, env, disease) → 风险评分
        │
        └──→ Treatment Agent
             ├── search_knowledge_base(query) → 语义检索结果
             ├── get_sop(disease, risk_level) → SOP 步骤
             └── generate_treatment_advice(disease, env) → AI 治疗建议
        │
        ▼
整合输出 → 前端渲染
```

## 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| Agent 框架 | LangGraph (Supervisor + StateGraph) | 2025+ |
| LLM | mimo-v2.5-pro | via OpenAI-compatible API |
| 工具定义 | @langchain_core.tools.tool | LangChain Core |
| 向量数据库 | ChromaDB | Latest |
| Embedding | BAAI/bge-small-zh-v1.5 | HuggingFace |
| 目标检测 | YOLOv8 | Ultralytics |
| Web 框架 | Flask | Latest |
| 气象 API | OpenWeatherMap | REST API |
| 状态持久化 | MemorySaver (Checkpointer) | LangGraph |

## 三种运行模式

| 模式 | 入口 | 特点 |
|------|------|------|
| 快速模式 | `run_diagnosis_pipeline()` | 直接函数调用 + LLM 报告，<5s |
| Supervisor 模式 | `run_agent_pipeline_with_trace()` | langgraph_supervisor 协调三个 ReAct Agent |
| StateGraph 模式 | `run_langgraph_diagnosis()` | 原生 StateGraph + Checkpointer 多轮对话 |

## 多轮对话架构

```
浏览器 (Session Cookie: thread_id)
        │
        ▼
Flask /api/langgraph_diagnosis
        │
        ▼
StateGraph.invoke(config={"thread_id": "session-xxx"})
        │
        ├─ Checkpointer: 自动加载该 thread_id 的历史 State
        ├─ treatment_node: 读取 conversation_history，注入 LLM prompt
        └─ update_history_node: 将本轮摘要追加到 history
        │
        ▼
Checkpointer: 自动保存更新后的 State
```
